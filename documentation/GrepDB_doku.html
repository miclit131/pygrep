PROJECT DESCRIPTION

The Unix command grep, known from Linux, served as the idea for this project. With grep, files can be searched for specific strings. The matching rows are the output.

Pygrep is a command line application that, similar to the grep command, can be used to search for specific strings and also numeric values. However, no text files are searched here, but databases. The value you are looking for is entered via the terminal. The columns of the database that contain the searched value are also displayed there. With different flags you can filter and limit the search.

These are presented and explained in more detail later in this documentation.

ARGUMENTS AND FLAGS - HOW TO USE PYGREP

Pygrep is a command line application. So to get started the terminal has to be open. Navigate the terminal to the directory where the script is located using the cd command.

Type “python pygrep.py” in the terminal to execute the script.

Searching a string in a specific database, table and column:

python pygrep.py search E6 -i Local_Postgres -t datamining -c Schadstoffklasse

argument

description

search

Represents the string/number that is searched for. It is always required. You can always enter just one value.

In this case it is “en”.

-i, –id-databases

The id of the database. It is the one that is entered in the database.yaml file.

It is not required. If the id is not specified the application will search in all of the databases.

You can also enter more than one ids if you want to search in specific databases.

-t, -table

-t is the table of a database.

If this argument is missing, the program will not work.

-c, –column

Represents the column of the given table.

When -c is not given, all columns of the table are searched in and only the columns that contain the searched string are presented in the output.

Besides the basic arguments there are also some flags:

-b, -e, -ex, -r

These four belong to a manually exclusive group. That means only one of them is allowed to use.

Example:

python pygrep.py search V -i Local_Postgres -t datamining -c HST Benennung -b

flag

description

-b, –begin

With -b you can search entries that begin with the searched input.

-e, –end

With -e you can search entries that end with the searched input.

-ex, –exact

With -ex you can search for only the data that contains exactly the input you entered.

The default search also outputs data that contain the searched string/number (regex).

-r, –range

With –range you can get the data between two values. It expects two values. The syntax here is a bit different. The search value here represents the column name and not the searched input. –range expects two values. The start value and the end value.

Another manually exclusive group contains some filtering options.

These options are helpful for big amounts of data so the output is clearer.

flags

description

-k, –keep-selected

With -k you can determine which columns are kept in the output.

It only works when a column (-c) is given.

-u, –unlimited-column

-u causes all columns in the output to be written next to each other.

-sc, –shortened-column

With -sc you can set the number of displayed columns. The default value is set to 5 if -sc is not specified.

-sr, –shortened-rows

With -sr you can set the number of displayed rows. (For example the first 10)

Sub Parser Manage

flag

description

-l, –list

-l lists all existing database entries

-n, –new-entry

add new database connection entry

-d, –delete

delete a existing database entry by entering name

python pygrep.py manage -del Example_ID

python pygrep.py manage -l

python pygrep.py manage -n

Examples

  • search for ‘en’ in database with id postgres in table datamining

python pygrep.py search en -i postgres  -t datamining

  • search in specific column

python pygrep.py search E6 -i postgres -t dataming -c Schadstoffklasse

  • search data between two values (prize between 30.000 and 60.000)

python pygrep.py search NeupreisBrutto -i postgres -t datamining -r 30000 60000

  • search a specific value

python pygrep.py search Volkswagen -i postgres -t datamining -c HSTBenennung -ex

  • set columns to unlimited and print the first 15 rows

python pygrep.py search en -t datamining -u -sr 15

  • set the number of displayed columns to 10

python pygrep.py search E6 -t datamining -c Schadstoffklasse -sc 10

  • showing the 10 first rows of data that begins with the input (M)

python pygrep.py search M -t datamining -c UTBenennung -sr 10
